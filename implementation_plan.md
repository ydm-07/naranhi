# Implementation Plan - 나란히 (Naran-hi) 웹앱 개발

본 계획서는 실시간 이메일 인증(회원가입 시), 비밀번호 기반 로그인, AI EBS 진도 관리, 실시간 공부 시간 및 상태 공유, 현직자 Q&A 게시판을 갖춘 교육 평등 웹 애플리케이션 **나란히 (Naran-hi)**의 개발 방안을 정의합니다.

---

## 1. Goal Description

* **목적**: 경제적 환경에 구애받지 않고 평등하게 공부할 수 있도록 AI와 인적 연결을 지원하는 웹서비스 개발.
* **아키텍처**: Node.js 등의 개발 환경이 없는 클라이언트 환경에서도 직접 브라우저에서 실행 가능한 **고성능 싱글 페이지 애플리케이션 (SPA)** 형태로 개발합니다.
  * 외부 백엔드 서버 없이 프론트엔드에서 직접 **Resend API**를 연동하여 실제 이메일 전송(OTP 인증번호)을 처리합니다.
  * 사용자의 이메일, 비밀번호 및 인증 상태는 브라우저의 로컬 스토리지(Local Storage)를 사용하여 관리합니다.
  * 개발 환경 설정이나 라이브러리 설치 없이 `index.html`을 브라우저에 여는 것만으로 모든 기능이 완벽히 작동합니다.

---

## 2. User Review Required

> [!IMPORTANT]
> **이메일 인증 기반 회원가입 및 비밀번호 저장**
> * 사용자는 회원가입 시 **이메일과 비밀번호**를 설정합니다. 
> * 입력한 이메일의 소유를 확인하기 위해 Resend API를 통해 인증 코드를 발송하고, 이를 확인해야 가입이 완료됩니다.
> * 프론트엔드 단독 환경(SPA)이므로, 사용자의 비밀번호와 계정 정보는 시연을 위해 브라우저의 **Local Storage**에 암호화되지 않은 형태(혹은 단순 해시)로 임시 저장됩니다.

> [!NOTE]
> **실제 이메일 발송 연동 (Resend API Key 입력)**
> * 브라우저에서 실제 메일을 보내기 위해 **Resend API**(`https://api.resend.com/emails`)를 활용합니다.
> * 앱 화면 내에 **[개발자/설정 모드]**를 제공하여 사용자가 본인의 Resend API Key를 입력하고 즉시 테스트할 수 있도록 디자인합니다. 
> * API Key가 없는 상태에서도 바로 테스트할 수 있도록 **[데모 모드(이메일 전송 시뮬레이션)]**를 병행하여 탑재할 예정입니다.

## 4. Proposed Changes

### [Naran-hi Core Frontend Project]

모든 코드는 `c:\Naranhi-app` 디렉터리에 작성됩니다.

#### [NEW] [index.html](file:///c:/Naranhi-app/index.html)
* **역할**: 애플리케이션의 뼈대를 제공하고 모든 SPA 뷰(View)를 정의합니다.
* **구성 뷰**:
  1. **Login/Signup View**: 이메일 및 비밀번호 입력. 회원가입 시 인증 코드 발송 및 확인 UI. 개발자/데모 설정 모드 포함.
  2. **Onboarding / AI Plan View**: 학년, 목표 과목, 성취 수준 입력 카드 및 AI 로드맵 자동 생성 애니메이션 화면.
  3. **Main Dashboard View**: 3개 핵심 탭을 포함한 다이어리 스타일 레이아웃.
     * **AI와 나란히 (학습 및 진도 관리)**: EBS 진도 체크리스트 트래커, 일일 할 일을 관리하는 **플래너(To-Do)**, 24시간 블록 형태의 하루 일정을 시각화하는 **타임테이블(Timetable)** 위젯.
     * **함께 나란히**: 원형 공부 타이머, 상태 변경 셀렉터, 실시간 스터디원 현황판.
     * **미래와 나란히**: 현직자 Q&A 게시판 및 로드맵 피드백 등록창.

#### [NEW] [style.css](file:///c:/Naranhi-app/style.css)
* **역할**: 앱의 핵심 비주얼(UI/UX Vibe)을 구현하는 스타일시트입니다.
* **디자인 토큰**:
  * 배경: 따뜻하고 깨끗한 화이트 계열 (`#FCFCFA`)
  * 액센트 컬러: 연한 파스텔 블루 (`#E8F0FE`), 연한 파스텔 옐로우 (`#FFF9E6`), 포인트 네이비 (`#1A365D`)
  * 보더 및 모서리: 둥근 모서리 (`border-radius: 16px` 이상) 및 소프트 드롭 섀도우.
  * 애니메이션: 플래너 체크박스 인터랙션, 타이머 재생/일시정지 마이크로 인터랙션, 로드맵 체크 시 완료 효과음/축하 애니메이션, 탭 전환 트랜지션.

#### [NEW] [app.js](file:///c:/Naranhi-app/app.js)
* **역할**: 비즈니스 로직과 세션 상태 관리를 담당합니다.
* **주요 기능**:
  * **Auth Manager**: Resend API 연동 메일 발송 로직, 로컬스토리지 계정 관리.
  * **AI Roadmap & Planner Manager**: 맞춤형 EBS 주차별 강의 목록 빌드, 플래너(To-Do) 아이템 추가/삭제/완료 상태 토글, 타임테이블 블록 렌더링 로직.
  * **Live Timer Engine**: `setInterval`을 이용한 공부 타이머 및 가상 그룹 멤버들의 상태 시뮬레이션 루프.
  * **Q&A System**: 질문글 등록 및 현직자 멘토의 Mock 답변 피드백 자동 생성 로직.

## 5. Verification Plan

### Automated Tests
* 프론트엔드 단독 실행 파일이므로 브라우저 개발자 도구(Console) 상에서 JS 에러 시뮬레이션을 위한 에러 발생 여부를 확인합니다.
* 이메일 API 호출 시 HTTP Status Code가 200(성공)을 반환하는지 모니터링합니다.

### Manual Verification
1. **플래너 및 타임테이블 테스트**:
   * 플래너에 새로운 항목을 추가하고 체크박스를 누를 때 취소선이 정상적으로 그어지는지 확인.
   * 타임테이블 위젯이 시간대에 맞게 시각적으로 잘 렌더링되는지 확인.
2. **이메일 인증 및 비밀번호 로그인 테스트**:
   * 가입된 이메일과 비밀번호로 로그인이 성공하는지 확인.
3. **AI EBS 로드맵 테스트**:
   * 설문 입력 후 생성된 EBS 로드맵이 주차별로 정상 출력되는지 확인.
4. **타이머 및 그룹원 상태 공유 테스트**:
   * 가상 스터디 멤버들이 주기적으로 상태를 업데이트하며 학습 자극을 주는지 모니터링.
